= Typed-WASM Multi-Producer Coordination Ledger
:toc: macro
:toclevels: 3
:icons: font

[IMPORTANT]
====
This is the *cross-repo coordination ledger* mandated by ADR-021
(accepted 2026-05-24).  It is the *single serialised queue* through
which ABI-touching typed-wasm proposals land.  Without it, parallel
proposals from different producers would race and the four-axis
model collapses to ad-hoc Refs trails — exactly the failure mode
ADR-021 prevents.

This ledger is *AffineScript's local copy* of the coordination
state.  Sister repos (`hyperpolymath/typed-wasm`,
`hyperpolymath/ephapax`) hold mirrors.  All three are *expected to
agree*; if they disagree, the typed-wasm repo's copy is canonical
(per ADR-021 conflict-resolution axis 4: queue ownership lives with
the consumer authority).

Machine-readable companion:

====

== The four axes (reminder from ADR-021)

. *Spec authority* — `lib/tw_verify.ml` (OCaml) today; flips to
  `hyperpolymath/typed-wasm` Rust verifier crate on C5.1 closure.
. *Coordinated landing protocol* — ABI-touching ADRs mirrored
  across all three repos; verifier ships parse-support FIRST;
  producers flip emit together; deprecation window ≥ one
  coordinated cycle.
. *Test parity protocol* — each ABI change requires fixtures
  contributed to the cross-compat suite (INT-12 / C5.1) from each
  producer.
. *Conflict resolution* — in-flight ABI proposals MUST be
  serialised; the queue is owned by `hyperpolymath/typed-wasm`.

== Queue conventions

Each queue entry has:

* *Slot* — short ID, e.g. `Q-001`.
* *ADR pair* — the ADR in this repo and the mirror ADRs in the
  sister repos.
* *State* — one of `queued`, `verifier-parse-shipping`,
  `producer-emit-flipping`, `landed`, `deprecation-window`,
  `closed`.
* *Owner* — the human/team driving the landing across repos.
* *Cross-compat fixtures* — the corpus contribution required for
  test parity.
* *Deprecation* — what version(s) the landing supersedes, and the
  removal date.

A proposal at `queued` is paused, not refused.  Promotion to the
next state requires explicit movement (not silent drift).

== State machine

[source]
----
queued
  │
  │ ADR ratified in all three repos
  ▼
verifier-parse-shipping
  │
  │ verifier-parse PR merged in this repo + typed-wasm + ephapax
  ▼
producer-emit-flipping
  │
  │ producer-emit PR merged in this repo + ephapax (atomic window)
  ▼
landed
  │
  │ ≥ one coordinated cycle elapses
  ▼
deprecation-window
  │
  │ verifier drops parse-support for the deprecated version
  ▼
closed
----

A slot may regress (e.g. `verifier-parse-shipping` →
`verifier-parse-shipped-here-pending-elsewhere`) only by *explicit
ledger entry*, never by silent drift.

== Rules

1. *No parallel ABI proposals.*  At most one slot is in any of the
   active states (`verifier-parse-shipping`,
   `producer-emit-flipping`, `landed`,
   `deprecation-window`) at a time.  Additional proposals enter
   `queued`.
2. *Local copies must agree.*  If this ledger disagrees with the
   typed-wasm-repo mirror, the typed-wasm copy is canonical.
3. *Mirror discipline.*  Every promotion of state in any repo's
   copy is mirrored in the other two within one working day.
4. *Verifier-first rule.*  No producer-emit PR may land in *any*
   repo before *all* repos have shipped verifier-parse-support.
5. *Cross-compat parity.*  No promotion to `landed` may occur
   without the fixture corpus from each producer being updated.
