= Categorical Semantics of AffineScript

*Document Version*: 1.0 *Last Updated*: 2024 *Status*: Theoretical
framework complete

== Abstract

This document provides a categorical semantics for AffineScript,
interpreting the type system in suitable categorical structures. We
construct models using: 1. Locally Cartesian closed categories (LCCCs)
for dependent types 2. Symmetric monoidal categories for linear/affine
types 3. Graded comonads for quantitative types 4. Freyd categories for
effects 5. Fibrations for refinement types

== 1. Introduction

Categorical semantics provides: - A mathematical foundation independent
of syntax - Proof of consistency and relative consistency - Guidance for
language extensions - Connection to other mathematical structures

AffineScript requires multiple categorical structures due to its
combination of features.

== 2. Preliminaries

=== 2.1 Category Theory Basics

*Definition 2.1 (Category)*: A category C consists of: - Objects: ob(C)
- Morphisms: C(A, B) for objects A, B - Identity: id++_++A : A вҶ’ A -
Composition: g вҲҳ f : A вҶ’ C for f : A вҶ’ B, g : B вҶ’ C

satisfying identity and associativity laws.

=== 2.2 Key Categorical Structures

*Cartesian Closed Category (CCC)*: - Terminal object 1 - Binary products
A Г— B - Exponentials B^A (function spaces)

*Locally Cartesian Closed Category (LCCC)*: - Each slice category C/О“ is
CCC - Models dependent types

*Symmetric Monoidal Category (SMC)*: - Tensor product вҠ— - Unit object I
- Associativity, commutativity (up to isomorphism)

*Symmetric Monoidal Closed Category (SMCC)*: - SMC with internal hom A вҠё
B - Models linear types

== 3. Interpretation of Types

=== 3.1 Base Types

Interpret types as objects in category C:

....
вҹҰUnitвҹ§ = 1                    (terminal object)
вҹҰBoolвҹ§ = 1 + 1                (coproduct)
вҹҰNatвҹ§ = N                     (natural numbers object)
вҹҰIntвҹ§ = Z                     (integers)
....

=== 3.2 Function Types

*Simple Functions* (in a CCC):

....
вҹҰП„ вҶ’ Пғвҹ§ = вҹҰПғвҹ§^вҹҰП„вҹ§
....

*Linear Functions* (in a SMCC):

....
вҹҰП„ вҠё Пғвҹ§ = вҹҰП„вҹ§ вҠё вҹҰПғвҹ§
....

*Effectful Functions* (in Freyd category):

....
вҹҰП„ вҶ’{Оө} Пғвҹ§ = вҹҰП„вҹ§ вҶ’ T_Оө(вҹҰПғвҹ§)
....

where T++_++Оө is the monad/applicative functor for effect Оө.

=== 3.3 Product Types

....
вҹҰП„ Г— Пғвҹ§ = вҹҰП„вҹ§ Г— вҹҰПғвҹ§           (Cartesian product)
вҹҰП„ вҠ— Пғвҹ§ = вҹҰП„вҹ§ вҠ— вҹҰПғвҹ§           (Tensor product, linear)
....

=== 3.4 Sum Types

....
вҹҰП„ + Пғвҹ§ = вҹҰП„вҹ§ + вҹҰПғвҹ§           (Coproduct)
....

=== 3.5 Record Types (Row Polymorphism)

Records are interpreted as dependent products over finite label sets:

....
вҹҰ{lвӮҒ: П„вӮҒ, ..., lвӮҷ: П„вӮҷ}вҹ§ = вҲҸ_{i=1}^n вҹҰП„бөўвҹ§
....

With row polymorphism:

....
вҹҰ{l: П„ | ПҒ}вҹ§ = вҹҰП„вҹ§ Г— вҹҰ{ПҒ}вҹ§
....

== 4. Dependent Types

=== 4.1 Locally Cartesian Closed Categories

For dependent types, we work in an LCCC C.

*Contexts as Objects*:

....
вҹҰВ·вҹ§ = 1
вҹҰО“, x:П„вҹ§ = ОЈ_{вҹҰО“вҹ§} вҹҰП„вҹ§        (dependent sum in slice)
....

*Types as Objects in Slice*:

....
вҹҰО“ вҠў П„вҹ§ вҲҲ ob(C/вҹҰО“вҹ§)
....

=== 4.2 О -Types

*Interpretation*:

....
вҹҰО (x:П„). Пғвҹ§ = О _П„(вҹҰПғвҹ§)
....

where О ++_++П„ is the right adjoint to pullback along П„ : вҹҰП„вҹ§ вҶ’ вҹҰО“вҹ§.

*Adjunction*:

....
ОЈ_П„ вҠЈ П„* вҠЈ О _П„

where:
ОЈ_П„ : C/вҹҰО“,x:П„вҹ§ вҶ’ C/вҹҰО“вҹ§     (dependent sum)
П„*  : C/вҹҰО“вҹ§ вҶ’ C/вҹҰО“,x:П„вҹ§     (weakening/pullback)
О _П„ : C/вҹҰО“,x:П„вҹ§ вҶ’ C/вҹҰО“вҹ§     (dependent product)
....

=== 4.3 ОЈ-Types

*Interpretation*:

....
вҹҰОЈ(x:П„). Пғвҹ§ = ОЈ_П„(вҹҰПғвҹ§)
....

=== 4.4 Identity Types

*Interpretation* (in a category with path objects):

....
вҹҰa == bвҹ§ = Path_П„(вҹҰaвҹ§, вҹҰbвҹ§)
....

where Path++_++П„ is the path object functor.

== 5. Quantitative Types

=== 5.1 Graded Comonads

Quantities are modeled by a graded exponential comonad:

*Definition 5.1*: A graded comonad on C indexed by semiring R is: -
Functors D++_++ПҖ : C вҶ’ C for each ПҖ вҲҲ R - Natural transformations: - Оө :
D++_++1 A вҶ’ A (counit) - Оҙ : D++_{++ПҖвӮҒ Г— ПҖвӮӮ} A вҶ’ D++_{++ПҖвӮҒ}(D++_{++ПҖвӮӮ}
A) (comultiplication) - Оё : D++_++0 A вҶ’ I (dereliction for 0) - c :
D++_++Пү A вҶ’ D++_++Пү A Г— D++_++Пү A (contraction for Пү) - w : D++_++Пү A вҶ’
I (weakening for Пү)

=== 5.2 Interpretation

....
вҹҰПҖ П„вҹ§ = D_ПҖ(вҹҰП„вҹ§)

вҹҰ0 П„вҹ§ = D_0(вҹҰП„вҹ§) вү… I          (erased)
вҹҰ1 П„вҹ§ = D_1(вҹҰП„вҹ§) вү… вҹҰП„вҹ§        (linear)
вҹҰПү П„вҹ§ = !вҹҰП„вҹ§                   (exponential comonad)
....

=== 5.3 Quantity Semiring Structure

The semiring laws correspond to:

....
D_0 вҲҳ D_ПҖ вү… D_0              (0 Г— ПҖ = 0)
D_ПҖ вҲҳ D_0 вү… D_0              (ПҖ Г— 0 = 0)
D_1 вҲҳ D_ПҖ вү… D_ПҖ              (1 Г— ПҖ = ПҖ)
D_ПҖвӮҒ вҲҳ D_ПҖвӮӮ вү… D_{ПҖвӮҒГ—ПҖвӮӮ}      (multiplication)
....

== 6. Effects

=== 6.1 Freyd Categories

Effects are modeled in a Freyd category (C, J, K): - C is a Cartesian
category (pure computations) - K is a category (effectful computations)
- J : C вҶ’ K is identity on objects, cartesian on morphisms

=== 6.2 Effect Algebra

Effects form an algebra of operations and equations:

*Definition 6.1 (Effect Theory)*: An effect theory is a Lawvere theory T
with: - Sorts (value types) - Operations: op : П„ вҶ’ Пғ - Equations between
terms

=== 6.3 Free Monad Interpretation

For an effect signature ОЈ:

....
вҹҰОөвҹ§ = Free_ОЈ
....

where Free++_++ОЈ is the free monad on the functor corresponding to ОЈ.

=== 6.4 Handler Interpretation

A handler for effect E is an E-algebra:

....
h : F_E(A) вҶ’ A
....

where F++_++E is the effect functor.

*Handle*:

....
вҹҰhandle e with hвҹ§ = fold(h, вҹҰeвҹ§)
....

=== 6.5 Effect Row Polymorphism

Effect rows are interpreted as colimits:

....
вҹҰОөвӮҒ | ОөвӮӮвҹ§ = вҹҰОөвӮҒвҹ§ вҠ• вҹҰОөвӮӮвҹ§
....

where вҠ• is coproduct of effect theories.

== 7. Ownership and Borrowing

=== 7.1 Presheaf Model

Model ownership using presheaves over a category of regions:

*Definition 7.1*: Let R be the category of regions with: - Objects:
Regions (lifetimes) - Morphisms: Inclusions вҖҷa вүӨ вҖҷb

A type with lifetime is a presheaf on R:

....
вҹҰref['a] П„вҹ§ : R^op вҶ’ Set
вҹҰref['a] П„вҹ§('b) = if 'a вүӨ 'b then вҹҰП„вҹ§ else вҲ…
....

=== 7.2 Affine Category

Ownership uses an affine symmetric monoidal category: - Objects: Types
with ownership annotations - Morphisms: Functions respecting ownership -
Tensor: Combines owned values (linear) - Weakening: Allows dropping
(affine, not linear)

=== 7.3 Borrow Semantics

Borrows are modeled as comonadic access:

....
вҹҰref П„вҹ§ = R(вҹҰП„вҹ§)              (reader comonad)
вҹҰmut П„вҹ§ = S(вҹҰП„вҹ§)              (state comonad, exclusive)
....

== 8. Refinement Types

=== 8.1 Fibrations

Refinement types are modeled in a fibration:

*Definition 8.1*: A fibration p : E вҶ’ B is a functor with cartesian
liftings.

For refinements: - B = types - E = refined types (types with predicates)
- p = forgetful functor

=== 8.2 Predicate Interpretation

....
вҹҰ{x: П„ | ПҶ}вҹ§ = {a вҲҲ вҹҰП„вҹ§ | вҹҰПҶвҹ§(a) = true}
....

As a subobject:

....
вҹҰ{x: П„ | ПҶ}вҹ§ вҶЈ вҹҰП„вҹ§
....

=== 8.3 Subset Types

Using subset types in a topos:

....
вҹҰ{x: П„ | ПҶ}вҹ§ = ОЈ_{a:вҹҰП„вҹ§} вҹҰПҶ(a)вҹ§
....

where вҹҰПҶ(a)вҹ§ is a proposition (subsingleton).

== 9. Soundness

=== 9.1 Interpretation of Terms

Each typing judgment is interpreted as a morphism:

....
вҹҰО“ вҠў e : П„вҹ§ : вҹҰО“вҹ§ вҶ’ вҹҰП„вҹ§
....

=== 9.2 Soundness Theorem

*Theorem 9.1 (Soundness)*: The interpretation is sound: 1. Well-typed
terms denote morphisms 2. Equal terms denote equal morphisms 3.
Reduction preserves denotation

*Proof*: By induction on typing derivations, verifying categorical
equations. вҲҺ

=== 9.3 Adequacy

*Theorem 9.2 (Adequacy)*: For closed terms of observable type:

....
вҹҰeвҹ§ = вҹҰe'вҹ§ implies e вүғ e' (observationally equivalent)
....

== 10. Coherence

=== 10.1 Coherence for Row Polymorphism

*Theorem 10.1*: Row-polymorphic terms have unique interpretations up to
canonical isomorphism.

The interpretation is independent of the order of record fields.

=== 10.2 Coherence for Effects

*Theorem 10.2*: Effect interpretations are coherent: different
derivations of the same typing judgment yield equal morphisms.

=== 10.3 Coherence for Quantities

*Theorem 10.3*: Quantity polymorphism is coherent: instantiation at
different quantities (respecting constraints) yields consistent
behavior.

== 11. Parametricity

=== 11.1 Relational Interpretation

Define relations over the categorical model:

*Definition 11.1*: For types П„, the relational interpretation вҹҰП„вҹ§++_++R
is: - вҹҰОұвҹ§++_++R = R (a relation, the parameter) - вҹҰП„ вҶ’ Пғвҹ§++_++R =
++{++(f, g) ++|++ вҲҖ(a,b) вҲҲ вҹҰП„вҹ§++_++R. (f a, g b) вҲҲ вҹҰПғвҹ§++_++R} - вҖҰ

=== 11.2 Parametricity Theorem

*Theorem 11.1 (Parametricity)*: For any polymorphic term
`О“ вҠў e : вҲҖОұ. П„`:

....
вҲҖ types A, B. вҲҖ relation R вҠҶ A Г— B.
(вҹҰeвҹ§(A), вҹҰeвҹ§(B)) вҲҲ вҹҰП„вҹ§_R[R/Оұ]
....

=== 11.3 Free Theorems

From parametricity, we derive free theorems:

*Example*: For `f : вҲҖОұ. List++[++Оұ++]++ вҶ’ List++[++Оұ++]++`:

....
вҲҖ g : A вҶ’ B. map g вҲҳ f_A = f_B вҲҳ map g
....

== 12. Models

=== 12.1 Set-Theoretic Model

The simplest model uses Set: - Types as sets - Functions as
set-theoretic functions - Effects as free monads

=== 12.2 Domain-Theoretic Model

For recursion, use CPO (complete partial orders): - Types as CPOs -
Functions as continuous functions - Recursion as least fixed points

=== 12.3 Topos Model

For full dependent types, use a topos: - Types as objects in a topos -
Dependent types in slice topoi - Refinements as subobjects

=== 12.4 Realizability Model

For extraction to computable functions: - Types as assemblies -
Functions as realized by programs - Connects to extraction

== 13. Examples

=== 13.1 State Monad

The state effect S++[++Пғ++]++ is modeled by:

....
вҹҰП„ вҶ’{State[Пғ]} ПҒвҹ§ = Пғ Г— вҹҰП„вҹ§ вҶ’ Пғ Г— вҹҰПҒвҹ§
....

State transformers.

=== 13.2 Linear Function Space

The linear function space:

....
вҹҰП„ вҠё Пғвҹ§ = вҹҰП„вҹ§ вҠё вҹҰПғвҹ§
....

where вҠё is the internal hom in a SMCC.

=== 13.3 Dependent Sum

The dependent sum:

....
вҹҰОЈ(x:П„). Пғвҹ§ = ОЈ_{a вҲҲ вҹҰП„вҹ§} вҹҰПғвҹ§(a)
....

A dependent pair (a, b) where b : Пғ++[++a/x++]++.
