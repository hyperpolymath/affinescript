== What an "evangelist" is

A tool (and ultimately an agent) that *finds, validates, demonstrates, and
advances* next-gen-language adoption across the estate. It automates the
pipeline a human/agent otherwise runs by hand for every kernel:

----
.res kernel
   │  ① triage        (migratable now? gated on which compiler wall?)
   ▼
re-decomposed .affine
   │  ② weak-points   (silent clamps? unguarded decoders? unbounded scans?)
   │  ③ boundary-proof (is the host->int encoding lossless, or a declared clamp?)
   │  ④ parity         (does the wasm match an independent oracle?)
   ▼
guide-linked readiness report
----

== The consistency the toolkit surfaces

`affine-assail` flags SecurityRank's clamp (PA-AFF-001), while
`echo-boundary` reports the in-band table `{Open:0..Strong:3}` as LOSSLESS
*and* the same table **with** the clamp (`+{OOB:0}`) as CONTROLLED LOSS.
That is not a contradiction — it is the whole point: the toolkit *flags*
the undeclared clamp and *lets you discharge it*, either by certifying the
lossless band or by declaring the controlled loss (the `@boundary`
obligation from `proposals/toolchain/` Finding 2). Flag -> certify, in one
pipeline.
